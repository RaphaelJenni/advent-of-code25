# Advent of Code 2025

## Guidelines
- **Separate Files**: Create a separate file/package for each day's task.
- **Tests**: Add tests for each solution (using the example data). Use kotest framework.
- **Utilities**: Create and use utility classes for recurring jobs (e.g., input reading).
- **Clean Code**: Adhere to clean code principles.
- **Naming**: Use speaking variable names (no abbreviations).

## Decisions
- **Language**: Kotlin
- **Package Structure**: `ch.rjenni.aoc2025.dayXX`
- **Input Storage**: `src/main/resources/dayXX.txt`
